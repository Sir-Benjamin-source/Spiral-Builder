# Spiral-Builder

**The compounding and embodiment layer of the Spiral Ecosystem**

Spiral-Builder is the official entry point and packaging layer for the Spiral framework. It provides clean, consistent ways to:

- Embody (install and initialize) the full Spiral ecosystem or individual components
- Create ready-to-use agent templates (especially for lightweight OpenClaw-style agents)
- Compound multiple Spiral modules into coherent, working instances
- Manage configuration, continuity, and reciprocity settings

## Purpose

While the core Spiral repos (`spiral-theory-core`, `Spiral-Path`, `spiral-recap`, `The-Spiral-Codex`, etc.) focus on specific capabilities, **Spiral-Builder** orchestrates them into usable, embodied forms.

It is designed to be:
- **Modular** — install only what you need
- **Approachable** — simple commands and clear defaults
- **Respectful** — encourages reciprocity, transparency, and good-faith use

## Quick Start

```bash
# Clone and embody the full ecosystem (recommended for most users)
git clone https://github.com/Sir-Benjamin-source/Spiral-Builder.git
cd Spiral-Builder
python -m embody --mode full
```

Or for a minimal setup:
```bash
python -m embody --mode light
```

## Embody Modes

- **`--mode light`** — Core theory + Spiral-Path only (fastest for experimentation)
- **`--mode full`** — Complete ecosystem including recap, codex, and lighthouse
- **`--mode custom`** — Choose individual components interactively

## What You Get

After embodying:
- Local Spiral-Codex with mycelial indexing
- Reciprocity checking (GenerosityExponent)
- Continuity via spiral-recap
- Optional SentinelAct legal shields
- Ready-to-use agent templates

## Philosophy

Spiral-Builder aims to make advanced reasoning tools accessible while maintaining the core values of the Spiral ecosystem:
- Coherence over speed
- Reciprocity between humans and AI
- Transparency and provenance
- Protection against misuse
